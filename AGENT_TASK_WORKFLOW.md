# Работа с задачами от ассистента

Этот документ описывает, как обрабатывать issues и локальные изменения, подготовленные ассистентом.

## 1. Проверка задачи
- Найдите issue в GitHub и убедитесь, что оно относится к нужному репозиторию и ветке.
- Изучите описание, примеры файлов и команды. Если в репозитории есть файлы с инструкциями (например, `AGENTS.md`), обязательно прочитайте их.

## 2. Подготовка рабочей ветки
1. Обновите основную ветку (`main` или целевую):
   ```bash
   git checkout main
   git pull
   ```
2. Создайте новую ветку для работы:
   ```bash
   git checkout -b feature/<краткое-описание>
   ```

## 3. Внесение изменений
- Реализуйте правки согласно описанию issue.
- Проверяйте стиль и качество кода с помощью линтеров и тестов, если они указаны.
- Регулярно просматривайте подсказки ассистента (команды, примеры, ссылки на файлы).

## 4. Подготовка коммита
1. Убедитесь, что подготовлены только нужные файлы:
   ```bash
   git status
   ```
2. Добавьте их в коммит:
   ```bash
   git add <файлы>
   ```
3. Создайте коммит с понятным сообщением:
   ```bash
   git commit -m "Краткое описание изменений"
   ```

## 5. Публикация результата
- Запушьте ветку в удалённый репозиторий:
  ```bash
  git push -u origin feature/<краткое-описание>
  ```
- Создайте Pull Request, опишите выполненную работу, как тестировали изменения и есть ли побочные эффекты.

## 6. Обратная связь и доработки
- Если при выполнении возникли вопросы, задавайте их ассистенту в issue или комментариях к PR.
- После ревью внесите правки, обновите коммит и снова запушьте ветку (`git commit --amend` + `git push --force-with-lease` или дополнительные коммиты).

Следуя этому процессу, можно эффективно сотрудничать с ассистентом и поддерживать качество репозитория.
